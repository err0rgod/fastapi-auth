# FastAPI-Auth Utility Roadmap (v1.x)
JWT support
PASETO support
Refresh token rotation
Role/permission management
OAuth providers (Google, GitHub, Discord)
Magic links
OTP/TOTP
Multi-tenant support
FastAPI integration
Good middleware/dependency injection APIs

## Phase 1: Core Utility Enhancements
- [ ] **Algorithm Agility:** Support alternative hashing algorithms (Bcrypt, Scrypt) alongside Argon2.
- [ ] **JWT Flexibility:** Support Asymmetric signing (RS256, EdDSA) instead of just HMAC (HS256).
- [ ] **Improved Rate Limiting:** Add "Sliding Window" strategy in addition to the current "Fixed Window" counter.
- [ ] **Enhanced Validators:** Add specialized validators for Email structure and MFA (TOTP) seed generation.

## Phase 2: Framework Integration & DX
- [ ] **FastAPI Dependencies:** Create pre-built FastAPI dependencies for easier token extraction.
- [ ] **Flask/Django Wrappers:** provide lightweight adaptors for other popular WSGI/ASGI frameworks.
- [ ] **Comprehensive Examples:** Create a `examples/` folder showing integration with Redis (Rate Limit), Postgres (Passwords), and MongoDB.
- [ ] **Type Stubs:** Ensure 100% type hint coverage for better IDE support.

## Phase 3: Advanced Security Patterns
- [ ] **MFA Core:** Add utility functions for TOTP (Google Authenticator) verification.
- [ ] **OAuth2/OIDC Helper:** Utilities for handling common OAuth2 flow artifacts (state, nonce).
- [ ] **Token Metadata:** Add helpers to extract device/OS info from User-Agents to help users with their own session tracking.
