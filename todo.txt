1. Refresh Token Rotation Helpers
      - Generate raw refresh token.
      - Hash refresh token for storage.
      - Verify submitted refresh token against stored hash.
      - Return metadata like expires_at, jti, family_id.
      - Keep storage decisions outside the library.

  2. Token Revocation / Blacklist Interface
      - Instead of isBlacklisted: bool, accept a callback:

  def is_revoked(payload: dict) -> bool:
      ...

  - Then require_auth() stays database agnostic and users can check Redis, SQL, MongoDB, etc.

  3. JWT Claims Support
     Add optional standard claims:
      - iss issuer
      - aud audience
      - scope / scopes
      - roles
      - permissions
      - nbf
      - custom claims

  This is very useful for real applications and still storage-free.

  4. Password Policy Object
     Instead of only a decorator, expose reusable validation:

  validate_password(password, policy=PasswordPolicy(...))
  validate_username(username, policy=UsernamePolicy(...))

  Let users customize min length, required symbols, allowed username chars, etc.

  5. Password Hash Upgrade Detection
     Argon2 supports checking if a hash needs rehashing. Add:

  needs_password_rehash(hash) -> bool
  verify_and_rehash(password, hash) -> tuple[bool, str | None]

  This helps apps upgrade security settings without forcing password resets.

  6. Storage Protocols
     Define lightweight protocols/interfaces for storage instead of requiring a specific DB:

  class RateLimitStore(Protocol):
      def incr(self, key: str) -> int: ...
      def expire(self, key: str, seconds: int) -> None: ...

  This makes your “database agnostic” claim stronger.

  7. Framework-Neutral Bearer Token Extraction
     Helpers like:

  extract_bearer_token("Bearer abc.def.ghi")

  Useful across FastAPI, Flask, Django, Starlette, Litestar, etc.

  8. Security Exceptions
     Replace generic ValueError with library exceptions:

  TokenlyError
  InvalidTokenError
  ExpiredTokenError
  RevokedTokenError
  RateLimitExceeded
  InvalidCredentialsError
  AccountLockedError

  This makes downstream apps cleaner.

  9. Rate Limit Result Object
     Instead of only returning count or raising, return:

  RateLimitResult(
      allowed=True,
      count=3,
      remaining=7,
      reset_after=42,
  )

  Then users can build headers like X-RateLimit-Remaining.

  10. Email / OTP Utility Layer
     Keep it database agnostic:

  - Generate numeric OTP
  - Hash OTP
  - Verify OTP
  - Check expiration
  - No email provider integration
